# Archflow — Чек-лист оставшихся работ

**Статус:** MVP развёрнут на https://archflow-app.netlify.app
**Дата:** 22 марта 2026

---

## Приоритет 1: Критично для продакшна

### Безопасность и стабильность
- [ ] **Email-верификация при регистрации** — сейчас confirm=true автоматом, нужен реальный email flow
- [ ] **Восстановление пароля** — страница "Забыли пароль?" с email-ссылкой
- [ ] **Rate limiting** — ограничение на API-запросы (защита от спама/DDoS)
- [ ] **HTTPS-only cookies** — убедиться что Supabase session не утекает
- [ ] **Sanitization inputs** — XSS-защита пользовательского ввода (комментарии, названия)
- [ ] **Error tracking** — Sentry или аналог для отлова ошибок в проде

### Данные и надёжность
- [ ] **Пагинация списков** — проекты, визиты, фото, документы (сейчас загружается всё)
- [ ] **Оптимистичные обновления** — UI обновляется сразу, откатывается при ошибке
- [ ] **Retry-логика** — автоматический повтор при сетевых ошибках
- [ ] **Кэширование** — SWR / React Query для stale-while-revalidate
- [ ] **Резервное копирование БД** — автоматические бэкапы Supabase

## Приоритет 2: Важно для пользователей

### Функционал
- [ ] **Удаление проекта** — кнопка в настройках с подтверждением
- [ ] **Удаление визита** — с каскадным удалением фото
- [ ] **Удаление фото** — кнопка на карточке фото
- [ ] **Удаление документа** — с очисткой Storage
- [ ] **Удаление участника** — из настроек проекта
- [ ] **Редактирование визита** — изменение даты, заметки
- [ ] **Редактирование счёта** — статус (оплачен), обновление ссылки
- [ ] **Статус визита** — ручной/автоматический (на основе фото)
- [ ] **Экспорт данных** — кнопка "Экспорт" (сейчас заглушка) → PDF-отчёт или Excel
- [ ] **Поиск и фильтрация** — по проектам, визитам, документам
- [ ] **Сортировка списков** — по дате, статусу, названию

### Уведомления
- [ ] **Push-уведомления (Web)** — Service Worker + FCM
- [ ] **Email-уведомления** — о новых замечаниях, просроченных счетах
- [ ] **Telegram-бот** — уведомления в Telegram (n8n-workflow уже подготовлен)
- [ ] **Настройки уведомлений** — сохранение в БД (сейчас UI-заглушка)
- [ ] **Отметка "прочитано"** — клик по уведомлению = прочитано

### Платежи
- [ ] **Статус оплаты** — кнопка "Оплачено" на счёте (manual toggle)
- [ ] **Авто-статус "Просрочен"** — cron-job или DB trigger при due_date < now()
- [ ] **Платёжные ссылки** — интеграция с ЮKassa / Тинькофф (генерация ссылок)

## Приоритет 3: UX-улучшения

### Интерфейс
- [ ] **Скелетон-загрузка** — вместо спиннера показывать контуры контента
- [ ] **Множественный toast** — очередь уведомлений (сейчас показывается только одно)
- [ ] **Подтверждение действий** — "Вы уверены?" перед удалением / сменой статуса
- [ ] **Drag-and-drop сортировка** — порядок этапов, позиций supply
- [ ] **Batch-загрузка фото** — загрузка нескольких фото одновременно
- [ ] **Прогресс загрузки** — полоска при загрузке файлов
- [ ] **Infinite scroll** — подгрузка при скролле вместо пагинации
- [ ] **Breadcrumbs улучшенные** — клик назад сохраняет scroll position
- [ ] **Dark mode** — переключатель темы (CSS variables уже частично готовы)

### Accessibility (a11y)
- [ ] **ARIA-атрибуты** — role, aria-label на модалках, табах, кнопках
- [ ] **Keyboard navigation** — Tab order, focus management при открытии модалок
- [ ] **Focus trap** — в модалках фокус не уходит за пределы
- [ ] **Screen reader** — alt-тексты на иконках, описания статусов
- [ ] **Color contrast** — проверка контрастности текста на цветных фонах
- [ ] **Не только цвет** — иконки + текст дополнительно к цветовой индикации статусов

### Mobile
- [ ] **PWA** — manifest.json, service worker, "Добавить на главный экран"
- [ ] **Offline-режим** — кэширование данных для просмотра без сети
- [ ] **Камера напрямую** — capture="environment" для фото с камеры на мобиле
- [ ] **Pinch-to-zoom** — на фотографиях в галерее
- [ ] **Swipe-навигация** — между вкладками проекта

## Приоритет 4: Масштабирование

### Архитектура
- [ ] **State management** — Zustand или React Context для глобального состояния
- [ ] **React Query / SWR** — замена кастомных хуков на проверенную библиотеку
- [ ] **Юнит-тесты** — Jest + React Testing Library для критических компонентов
- [ ] **E2E тесты** — Playwright для ключевых user flows
- [ ] **CI/CD** — GitHub Actions: lint → test → build → deploy
- [ ] **Мониторинг** — uptime, APM, логирование
- [ ] **CDN для файлов** — оптимизация загрузки фото/документов

### Бизнес-функционал
- [ ] **Мультиязычность (i18n)** — английский язык (для международных клиентов)
- [ ] **Шаблоны проектов** — создание проекта из шаблона (этапы, чек-лист)
- [ ] **Чек-листы визитов** — пункты проверки для каждого визита
- [ ] **Отчёты** — PDF-отчёт по визиту, проекту, комплектации
- [ ] **Интеграция с календарём** — Google Calendar / Яндекс.Календарь
- [ ] **Комментарии к фото** — нити обсуждений (threaded comments)
- [ ] **Версионирование документов** — история изменений, diff
- [ ] **Аналитика** — графики: расходы, сроки, эффективность
- [ ] **API для интеграций** — webhooks, REST API для внешних сервисов

### Коммерция
- [ ] **Тарифные планы** — Free / Pro / Business
- [ ] **Биллинг** — Stripe/ЮKassa подписки
- [ ] **Лимиты** — ограничение проектов/участников по тарифу
- [ ] **Онбординг** — интерактивный тур для новых пользователей
- [ ] **Landing page** — маркетинговая страница с ценами и фичами
